# خطة الهجرة — من الملف المصغّر إلى Next.js

الهدف: استبدال `public/index.html` المصغّر بتطبيق `web/` احترافي قابل للصيانة،
**دون انقطاع** للموقع الحالي. التبديل النهائي يتم فقط بعد تكافؤ الميزات.

## المبدأ
- النسخة القديمة (`/public`) تبقى هي الإنتاج حتى إشعار آخر.
- نبني الميزات في `web/` ميزةً تلو الأخرى، ونختبر، ثم نبدّل الدومين.

## الحالة

| الميزة | النسخة القديمة | إعادة البناء (`web/`) |
|---|---|---|
| نظام التصميم/الهوية (RTL، خطوط، ألوان) | ✅ (CSS مصغّر) | ✅ **منجز** |
| صفحة الهبوط (تسويق) | ✅ | ✅ **منجز** |
| صفحة المنيو العامة `/[slug]` (SSR) | ✅ (CSR) | ✅ **منجز** |
| SEO (metadata/OG/robots/sitemap) | ⚠️ جزئي | ✅ **منجز** (SSR أصيل) |
| PWA (manifest/SW) | ✅ | ✅ manifest / ⏳ SW |
| المصادقة (تسجيل/دخول) | ✅ | ✅ **منجز** (Supabase Auth + `@supabase/ssr` + proxy) |
| لوحة تحكم المطعم (CRUD قوائم/أصناف) | ✅ | ✅ **منجز** (onboarding + قوائم + أصناف: إضافة/تعديل/حذف/إظهار) |
| توليد QR للطاولات | ✅ | ✅ **منجز** (لكل طاولة + تنزيل PNG) |
| الإحصائيات (analytics) | ✅ | 🚧 لوحة قراءة (views/أقسام) — تتبّع حيّ لاحقاً |
| المدفوعات (Moyasar) | ✅ (pk_test) | ⏳ |
| المجلس الاستشاري AI (ai-proxy) | ✅ | ⏳ |
| الولاء (loyalty) | ✅ | ⏳ |
| المدونة (blog_posts) | ✅ | ⏳ (SSG قابل للأرشفة) |
| لوحة المؤسس (founder-admin) | ✅ | ⏳ |

## الترتيب المقترح للتنفيذ
1. ✅ **المصادقة** عبر `@supabase/ssr` (proxy + جلسات الكوكيز + دخول/تسجيل + حماية `/dashboard`).
2. ✅ **لوحة التحكم**: قوائم وأصناف (CRUD) — حقول الصنف من مصدر واحد type-safe (لا إسقاط صامت). يبقى: حقل `options` المركّب.
3. ✅ **QR + الإحصائيات**: مولّد QR لكل طاولة (PNG) + لوحة إحصائيات قراءة. يبقى: تتبّع المشاهدات الحيّ (جدول `analytics`).
4. **المدفوعات** (Moyasar) — مع الإبقاء على `pk_test` حتى الإطلاق. ← التالي
5. **AI / الولاء / المدونة / لوحة المؤسس**.

## ملاحظات تقنية
- الجداول وأنواعها موثّقة في `src/lib/types.ts` و`../CLAUDE.md`. قاعدة البيانات
  الحيّة في حساب Supabase منفصل (ref: `wjqpsbpebpntpeinqccl`) — حدّث الأنواع يدوياً
  عند تغيّر السكيمة، أو فعّل codegen عند توفّر صلاحية المشروع.
- **قاعدة الحقول الثلاثية** ما زالت تنطبق منطقياً: أي حقل جديد يجب أن يظهر في
  (الفورم + الإضافة + التحديث) — لكن هنا يصبح type-safe عبر TypeScript بدل
  الاستبدال النصّي.
- **الأمان:** أكواد المؤسس (`cm_fsecret`) وعمليات `founder-admin` حسّاسة —
  تُنقل خلف server actions / route handlers، لا في كود المتصفح.
