# خطة الهجرة — من الملف المصغّر إلى Next.js

الهدف: استبدال `public/index.html` المصغّر بتطبيق `web/` احترافي قابل للصيانة،
**دون انقطاع** للموقع الحالي. التبديل النهائي يتم فقط بعد تكافؤ الميزات.

## المبدأ
- النسخة القديمة (`/public`) تبقى هي الإنتاج حتى إشعار آخر.
- نبني الميزات في `web/` ميزةً تلو الأخرى، ونختبر، ثم نبدّل الدومين.

## الحالة

| الميزة | النسخة القديمة | إعادة البناء (`web/`) |
|---|---|---|
| نظام التصميم/الهوية (RTL، خطوط، ألوان) | ✅ (CSS مصغّر) | ✅ **منجز** |
| صفحة الهبوط (تسويق) | ✅ | ✅ **منجز** |
| صفحة المنيو العامة `/[slug]` (SSR) | ✅ (CSR) | ✅ **منجز** |
| SEO (metadata/OG/robots/sitemap) | ⚠️ جزئي | ✅ **منجز** (SSR أصيل) |
| PWA (manifest/SW) | ✅ | ✅ manifest / ⏳ SW |
| المصادقة (تسجيل/دخول) | ✅ | ✅ **منجز** (Supabase Auth + `@supabase/ssr` + proxy) |
| لوحة تحكم المطعم (CRUD قوائم/أصناف) | ✅ | ✅ **منجز** (onboarding + قوائم + أصناف: إضافة/تعديل/حذف/إظهار) |
| توليد QR للطاولات | ✅ | ✅ **منجز** (لكل طاولة + تنزيل PNG) |
| الإحصائيات (analytics) | ✅ | 🚧 لوحة قراءة (views/أقسام) — تتبّع حيّ لاحقاً |
| المدفوعات (Moyasar + مدى/Apple Pay/STC) | ✅ (pk_test) | ✅ **منجز** (نموذج Moyasar + اختيار الباقة) |
| المجلس الاستشاري AI (ai-proxy) | ✅ | ✅ **منجز** — عقد مؤكّد `{system,messages}`→`{text}` بـ JWT المستخدم |
| الولاء (loyalty) | ✅ | ✅ **منجز** (إعدادات + قائمة العملاء) |
| المدونة (blog_posts) | ✅ | ✅ **منجز** (قائمة + مقال، SSR/ISR قابل للأرشفة) |
| لوحة المؤسس | ✅ | ✅ **منجز** — لا توجد دالة founder-admin؛ لوحة تقرأ revenue/subs/tickets محميّة بـ FOUNDER_EMAIL |
| المدفوعات → تفعيل الاشتراك | ✅ (webhook) | ✅ **منجز** — نمرّر metadata{user_id,plan_id,cycle} فيفعّل moyasar-webhook الاشتراك |

## الترتيب المقترح للتنفيذ
1. ✅ **المصادقة** عبر `@supabase/ssr` (proxy + جلسات الكوكيز + دخول/تسجيل + حماية `/dashboard`).
2. ✅ **لوحة التحكم**: قوائم وأصناف (CRUD) — حقول الصنف من مصدر واحد type-safe (لا إسقاط صامت). يبقى: حقل `options` المركّب.
3. ✅ **QR + الإحصائيات**: مولّد QR لكل طاولة (PNG) + لوحة إحصائيات قراءة. يبقى: تتبّع المشاهدات الحيّ (جدول `analytics`).
4. ✅ **المدفوعات** (Moyasar) — نموذج دفع + اختيار باقة، `pk_test` حتى الإطلاق.
5. ✅ **AI / الولاء / المدونة / لوحة المؤسس** — منجزة (AI و founder بانتظار تأكيد عقد الـedge functions).

## حقائق مؤكّدة من قاعدة Cloudsmenu (introspection)
- **RLS مفعّل ومضبوط** على كل الجداول بنمط `auth.uid() = user_id` + قراءة عامة —
  تطبيقنا متوافق معه مباشرة، فلا حاجة لأي migration (حُذف الملف الزائد).
- **لا توجد دالة `founder-admin`**. الدوال الموجودة: `ai-proxy`, `moyasar-webhook`,
  `notify-support`, `dynamic-task`.
- **تنبيه تسعير:** `moyasar-webhook` يحسب الإيراد بأسعار `basic 49 / standard 99 /
  premium 199`، بينما واجهة الباقات تعرض `399/649/949`. **وحّد الأرقام** قبل الإطلاق
  (قرار تجاري — لم أغيّره).
- **ملاحظات أمان (advisors):** سياسات `USING(true)` على
  `announcements/promo_codes/revenue_log/support_tickets` فضفاضة — يُنصح بتشديدها.
  كذلك Buckets عامة تسمح بسرد الملفات، و«حماية كلمات المرور المسرّبة» معطّلة في Auth.

## المتبقّي لتكافؤ كامل
- تتبّع المشاهدات الحيّ (إدراج في `analytics` عبر beacon — ISR يمنع العدّ في SSR).
- حقل الأصناف المركّب `options` (نصّي)، وإدارة المدونة من اللوحة.
- ضبط Webhook في Moyasar Dashboard ليُشير إلى `moyasar-webhook`، وتوحيد الأسعار.
- تشديد سياسات RLS الفضفاضة + تفعيل حماية كلمات المرور المسرّبة في Auth.
